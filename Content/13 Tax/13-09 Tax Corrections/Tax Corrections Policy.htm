Tax Corrections Policy

  1. If a customer claims they are exempt, but they do not have an exemption certificate on file, it is required to collect tax.  The customer may return later with an exemption certificate and request a refund of tax collected. 
  2. Tax corrections must reference the correctly corresponding Tax Record Card. 
  3. Stores can only run tax corrections for sales originally processed at their location.
  4. Tax corrections must use the same rules used in sale transactions.  When using sales numbers in a correction transaction, the system will determine the taxability of each product.  When using the 9999-99998 number, users are required to correctly calculate the taxable amount and should review the taxability of each product based on the customer’s exemption reason before running the correction.
  5. Tax corrections are not to be used to refund tax to a customer disputing the taxability of product(s) on an invoice. These questions should be addressed with the Corporate Tax Department for review.
  6. Tax corrections are not to be used to adjust accounts which have short-payed their invoice due to a dispute over the taxable amount.  These questions should be addressed with the Corporate Tax Department for review.
  7. Stores incorrectly using the correction transaction to refund tax to a customer will be required to reverse the correction transaction.
  8. Customers may request refunds of collected tax for a certain period of time (typically, three to four years) which varies by state. Some certificates and permits are only retroactive for a certain time period and some may never allow for retroactive refund of tax. Carefully review customer documentation and state information to ensure compliance. 
    1. For example, the following states limit the time period in which sales tax may be refunded: Rhode Island, Michigan, Massachusetts, and Connecticut.  Refunds may be subjected to other limitations decreed by the state.
  9. Tax corrections must ALWAYS be completed using the correction transaction. NEVER refund tax as a line item on a return transaction.  While the customer’s accounting will balance, this results in lowered sales for the store.  Tax is not “sold” to a customer, but, “collected” for the state or local entity.  Refunding tax is a correction and never a return.
  10. When a customer requests a refund of tax over several invoices, please note that the tax correction transaction should tie back to an original transaction.  This means that a separate correction transaction must be run for each sale transaction run during the time period for which the customer requests a refund. 
  11. If you have 10 or fewer transactions, please handle each separately. If you have more than 10 transactions, run a single correction transaction en masse representing all invoices for which a refund is being granted.
  12. If the original transaction contained items which were taxed and items which were not taxed, the Correction Transaction should only be run for the product(s) requiring correction. 
    1. For example, if tax was charged incorrectly on two items, the Sub-Total should be entered as the extended Sub-Total of those two items. 
  13. Do not mix taxed and non-taxed items on the same Correction Transaction. If the correction requires both the collection and the refund of tax, it is required to run two separate transactions.
  14. If tax is not collected for a cash sale and it is later determined that tax should have been collected and the customer cannot be reached, the store must assume the tax liability for that sale. This may occur if the customer does not present or have on a file a tax exemption certificate at the time of the sale or if a tax exemption certificate on file is found to be invalid after the sale has been run.
  15. Sales number 9999-9999-8 is only to be used to correct sales tax. Do not use this number for any other purpose.
