Manager Approval of Time-Off Requests Procedure

Store management must approve or refuse all time-off requests. Store Managers, Assistant Managers, and Operations Managers will all be able to process these requests for other employees but only Store Manager requests will auto-approve.  Managers should approve / refuse all time-off requests as soon as is practical and before the pay period in which the request would be scheduled.

Step Action

1

Access Kronos using the Time Stamp button on the POS toolbar. Use your mySherwin credentials to log in.

2

Select Requests Alert Widget from the upper center of the screen. If you have requests to process, the ticker will alert you to the number of pending requests.  

Request notifications can also be found:

  • Requests (defaults to current pay period requests)
  • Inbox -> Messages

3

Select View All.

4

Select Time Off.

Note: The widget will default to showing you a rolling six months into the future.

5

Select the appropriate employee name/request column to highlight it for action.

6

If the request is for paid time off, select the accruals tab in the lower left corner to quickly validate remaining employee personal day and vacation allowances. 

Note: Kronos will allow users to request more vacation or personal days than their accruals will allow for; Kronos should warn you if you attempt to approve a request in excess of available accruals. Be sure to confirm that your employee has enough time to fulfill their request for a vacation, personal day, or local sick time.

7

Select the desired action (hover over actions for more information):

Action Notes
Details

Displays all information available on the main request screen as well as the Request Type (full day vs. half day vacation)
Edit

Do not use
Approve

Approve
Refuse

Refuse an employee time-off request.
Pending

No action. Do not use.
Retract

Do not use.
Request Time Off

Create and submit a paid time-off request on behalf of an employee.

Note: This entry will still need to be approved by management before considered scheduled.

Note: Any action on the request will prompt a notification to the employee’s Kmail inbox. The request alert button will reflect the number of pending requests (select the refresh button next to the alert symbols to update).

8

Did you select Approve or Refuse?

If Then
Approve

The procedure is complete.
Refuse
Step Action

1

Select Other – Enter Comment in Notes from the Comments Drop Down Box.

2

Enter reasons for the refusal in the Notes Box.  Limit comments to business reasons: inventory, staffing coverage in light of staffing need, available accruals.

3

Select Refuse.
