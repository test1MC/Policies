APO Codes Procedure

APO Codes are assigned to each product in the Master Product File and identify the source of a product as internal or external.  Products coded as external may only be ordered on an external purchase orders.  Products available internally (including vendor products stocked at the DSC) may only be ordered on internal purchase order types.  The APO Code also controls which products may be maintained for Min / Max order points.  Products which have been assigned an APO Code, which is allowable for STAR Orders, will also be eligible for Min / Max Maintenance.  If a product’s APO Code assignment does not allow the product to be ordered using STAR Orders, that product is not eligible for Min / Max Maintenance.

To review which APO Codes are associated with external purchase orders and those which are associated with internal purchase order types, stores should access the View Allowable Purchase Order (APO) Codes Report in Merchandise Management Reports.  This report will show the same information displayed in the chart below:

Stores may review an individual product's APO Code by selecting the INQUIRY [F4] option for Products and Prices. 

Step Action
If you receive the error message, "PRODUCT CANNOT BE ENTERED ON THIS TYPE OF ORDER," follow these steps. 
1 Check the APO Code on the product profile in INQUIRY [F4]  Products and Prices.
2 Cross-check this code against that on the Allowable Purchase Order (APO) Codes Report to verify that the product may be placed on the order type being used. 
3 Refresh the product information from the HQ database following the Product Refresh Procedure. 
4 Call the SCO Helpdesk at 216-566-2740 for assistance.
