Update NSF Bad Data Procedure

If the NSF Check email notification indicates there is “Bad Data,” log on to the NSF Check Management System to correct this information. Typically, this information cannot be machine-read at the bank and results in misconstrued name and/or address information.

Step Action

1

From SOURCE homepage -> Operations -> NSF Check Management System.

2
If Then
Updating Bad Debt Select Update a Bad Address.
Posting a returned check to for a cash account sale Select Add a New NSF Check.
Processing a payment Follow the Miscellaneous Cash Receipt Procedure.
The NSF check was for a payment on account transaction Refer the NSF check to the District Credit Office.

3

Locate and select the correct account from the list.

4

Update the missing or incorrect information.

Note: The NSF Check Management System pulls data from SOURCE and will update it accordingly.

5

Select Update.

Note: Select Receive Payment to record a payment for the customer.
