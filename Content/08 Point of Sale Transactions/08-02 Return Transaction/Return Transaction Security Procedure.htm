Return Transaction Security Procedure

Return Transaction Security is available if:

  • The transaction number is not valid.
  • The return amount is greater than the original sale.
  • The original form of payment was a credit card.
  • The return is processed within a 5-day period.

Return Transaction Security Scenarios

1

The employee processing the return is responsible for entering the date and initialing the returned item on the customer’s receipt (after crossing out the entry on the sales invoice).

Note: If customers do not want the original receipt, simply tear it up (show the customer that it has been destroyed).
2 From POS desktop, select Store -> Point of Sale -> Select Return from the Sale/Payment Menu .

3

The customer should have a receipt to return the merchandise. If the customer does not have the original receipt and the Manager has approved the return, press ACCEPT [F1]. Do not enter the terminal number if the transaction number is unknown.

The following error message will return to the screen if the transaction number is not valid: TRANSACTION NOT ON FILE…try again

4

The transaction may not be valid because the terminal/transaction numbers or the date were entered incorrectly.

5

The following error message will return to the screen if the transaction was not for a sale transaction: THIS TRANSACTION WAS NOT A SALE TRANSACTION…try again

6

You will not be permitted to process return transactions to a non-sale/correction transaction.

7

Customer information (name, address, city, state, zip code, phone, and customer signature) must be complete and accurate (no abbreviations) any time the customer receives cash, credit, or a store draft on a Cash Refund or Cash Correction Transaction.

8

The amount of the return may not exceed the original sale. This refers to any discounts, markdowns, or sale prices that may have been in effect when the original purchase was made.

The following warning message will return to the screen if the return amount is greater than the original sale: THIS AMOUNT EXCEEDS THE TOTAL FROM THE ORIGINAL SALE…press any key

9

A draft must be used if the cash refund is over $100. It must also be used if there is not enough cash in the drawer to cover the refund and to maintain sufficient operating funds.

10

If an original sale was made to the customer’s credit card, then the return must be credited to the credit card. Do not give cash back on a credit card return.

The following warning message will return to the screen for a refund transaction in which the original form of payment was a credit card: THIS REFUND SHOULD BE CREDITED TO THE CREDIT CARD USED FOR THE PURCHASE…do you want to continue? Y/N

11

If the original purchase was with check, allow fourteen (14) days for customer check to clear the bank before completing the refund. Some stores have the option to refund check payments as drafts or in SWeep. If customers request a check as the form of refund, inform them that a check will be mailed to them after the fourteen(14) day period. This only applies to stores located in the Continental U.S.

The following warning message will return to the screen if a return is processed within the 14-day period: [A CHECK WAS USED FOR THIS PURCHASE WITHIN THE LAST 14 DAYS…do you want to continue? Y/N]

12

If the customer’s check has not cleared the bank (10 days or less from the time of the purchase), process a Correction transaction. Return the merchandise on the [negative] side of the transaction and process it as a deposit on the [positive] side of the transaction. Press COMMENTS [F9] to add a note, “Deposit holding for clearing of customer’s check”. Select Cash as the form of refund. After the ten (10) day waiting period, process a Return transaction for the customer’s deposit holding. This process will allow you to return the merchandise to inventory and mail the check to customers after their check clears the bank. 

13

You are permitted to return customers’ checks if they request a complete refund the same day the purchase was made. Process a return for the total amount. Make a note in the comments area that you are returning the customer’s check. The form of refund will be cash. You are required to document your cash/check variance during the close process.
