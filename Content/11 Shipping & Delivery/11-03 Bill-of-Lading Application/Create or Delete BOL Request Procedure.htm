Create or Delete BOL Request Procedure

The Create Bill-of-Lading Request can be used to submit information to create a Bill-of-Lading for Customer Delivery Orders, Transfer Orders, Demonstration Orders, Customer Returns, and Customer Will Call Requests. Print a Pick Ticket copy of the customer order. Have the person picking and preparing the order manually note how the items have been packed on the pick ticket. Provide this pick ticket to the person creating the Bill-of-Lading for use in creating Repack Cartons.

The Delete Bill of Lading Request on the Bill of Lading Menu in SOURCE allows users to cancel a BOL request.  You may want to delete a BOL request if you have already submitted the information and need to add or remove products from the delivery.  Only requests, which are in Confirmed or Editing status in BOLDOC, may be deleted via SOURCE. Once a request is deleted, re-submit the data from SOURCE. A new HEARS ID will be created and you will need to begin the editing process in BOLDOC

Step Action

1

From POS desktop, select Store -> Customer Order Entry -> Delivery Menu -> Open BOLDOC.
2
If Then
Creating

Select Create Bill-of-Lading -> Proceed to Step 3.

Note: The Order Line Browse defaults to only display Customer Delivery Order Lines in Ready for Delivery Status. Use this to find Order Lines for Customer Delivery Orders, Store Expenses, and Transfer Orders. 

Deleting

Select Delete Bill-of-Lading Request -> Search for the request by entering Customer Account Number, Customer Order Number, Customer Order Control Number, or HEARS ID or use LOOKUP [F3] -> Select the BOLDOC request and Yes when prompted -> Procedure is complete.

Note: You cannot resubmit a BOL request for an IBAR.

3

Press Lookup [F3] to access the Lookup Options to browse for Order Lines for any order line not existing on a customer order marked for delivery.

4

Enter the customer account number or press Lookup [F3].

Note: If searching by Customer Account, query against the default browse, which will narrow the default browse display of Ready for Delivery orders.

5

Use the Up and Down keys to navigate and press Enter to include Order Lines on the BOL.

Note: An X displays in the S (Select) Column. Lines may be selected from several orders for a BOL but all orders must be associated with a Customer Account.

6

Press ACCEPT [F1] to save BOL.

Note: The Customer Order Control # displays in the most left column. Refer to this column to ensure that all required Order Lines for the request are selected.

7

Verify the ShipTo Address.

Note: If the address needs to be corrected, change the address for the Bill-of-Lading.  This will not change the address on the Customer Order, or transaction.

8

Press LOOKUP [F3] to search for the carrier.

Note: Only active carriers will display in this lookup and preferred carriers will display at the top of the listing.  If the Carrier has a default Shipping Terms value, that will automatically default in the BOL information screen. Otherwise, this will default to UNKNOWN and you will be required to edit the value in BOLDOC.

9

Verify that the Ship Date is correct.

Note: The Ship Date will always default to today's date, but, if the order is not being shipped or picked up today, it may be changed.

10

Press ACCEPT [F1] and select Y when prompted to send BOL Request to BOLDOC.

Note: If all the required information has been completed, BOLDOC will assign a HEARS ID for the information submitted.  The HEARS ID is similar to the order Control Number and can be used to locate the BOL in BOLDOC or to delete the request in BOLDOC from SOURCE, if required.
